
* clone: creates a duplicate of a repo locally.
* checkout: swapping to a seperate branch entirely.
  - -b: creates a new branch and switches to it.
* status: inspects the current state of the git repository.
* push: used to upload local repository content to a remote repository.
  - -u: used to link a local branch to a specific remote branch.
* pull: updates your local project with changes from a remote server.
* add: move changes from your working directory to the staging area.
* commit: a snapshot of your project's files at a specific point in time, acting as a "save point" in your repository's history.
  - -m: save a snapshot of your staged changes to the local repository with a specific message..
  - -a: shortcut that automatically stages every file that is already tracked and has been modified or deleted, then performs the commit in a single step.
  - -am: a combined command that allows you to stage modified files and commit them with a message in a single step.
* branch: a lightweight, movable pointer to a specific commit.
  - -a: used to list all branches in your repository, including both local and remote-tracking branches. 
  - -d: used to safely delete a local branch in Git.
* merge: the process of integrating changes from one branch into another.
